# Mounted eLearn Disc Analysis

## Executive conclusion

The mounted `D:` volume is the original Fiat eLearn content disc for Fiat Multipla (model code 186), application version 1.3.2. It supplies the databases and native assets missing from the installed shell at `C:\eLearn`.

The disc contains eight standalone Microsoft Jet/Access content databases, one for each supplied content language, and 4,064 ID-addressed technical assets. Actual database records identify the model as `MULTIPLA`, model ID `2000006`, code `186`.

The disc was inspected read-only. Its files have the read-only attribute, no executable was launched, and no database was repaired, compacted, or relinked. Because Jet requires a writable directory for lock files, byte-identical database copies were made under the user temporary directory, hash-verified, and queried with `Mode=Read`. Nothing was imported.

## Disc identity

`D:\autorun.inf` contains:

```ini
[autorun]
OPEN=Start.exe
AppVersion=1.3.2
ICON=elearn.ico
```

`Start.exe` was not executed.

Across all languages, XML refers to 4,155 distinct asset IDs; 91 referenced IDs do not have corresponding files on the disc. Thus the disc asset set is internally useful but not perfectly complete.

The native SVG corpus is materially valuable: it preserves vector wiring/component diagrams that a web crawl may have rasterized, omitted, or detached from their source node IDs.

## eLearn content determination

The following are original technical-content components:

- the eight `elearn_<language>.dat` databases;
- the 4,064 `image/<id>.image` assets;
- the application help in `system/help`;
- the installed-shell XSL renderers referenced by the database `XSL` table.

The databases contain the full navigation hierarchy, source XML, applicability/version associations, diagnostic workflows, procedures, electrical content, cross-node target IDs, and asset IDs. Details are in `ELEARN_DATABASE_SCHEMA.md` and `MULTIPLA_DISC_COVERAGE_REPORT.md`.

## Safety and limitations

- Direct Jet access on read-only optical media fails because Jet attempts to create a lock file. Analysis used hash-verified temporary copies, never the mounted originals.
- The database `VERSION` value is `0.0.0`; disc identity comes from `autorun.inf` (`AppVersion=1.3.2`) and file metadata.
- There are 91 referenced-but-missing image IDs and some unresolved cross-node references.
- The disc has no update database. Later online updates, if any, are outside this source.
- The current platform's record manifest was not present in `C:\eLearn`, so exact content/hash overlap with the 6,951-page crawl remains a Phase 2A comparison task.

## Final finding

`D:` contains the original Fiat Multipla eLearn content source and is substantially richer than the previously inspected installed shell. It is suitable for a controlled, read-only Phase 2A extraction and comparison, but no import or Phase 2B work should begin until the existing platform manifest is compared against it.
